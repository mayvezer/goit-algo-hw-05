# Аналіз ефективності алгоритмів пошуку підрядка

## Результати тестування
На основі вимірювань за допомогою модуля `timeit` були отримані наступні результати (середній час за 10 ітерацій):

### Стаття 1
* **Боєр-Мур:** Найшвидший для існуючого підрядка.
* **Кнута-Морріса-Пратта:** Показав середній результат.
* **Рабін-Карп:** Найповільніший через обчислення хешів.

### Стаття 2
* **Боєр-Мур:** Знову продемонстрував найкращу швидкість, особливо на великих текстах.
* **Кнута-Морріса-Пратта:** Ефективний, але поступається Боєру-Муру.

## Загальний висновок
Найшвидшим алгоритмом у більшості випадків виявився **алгоритм Боєра-Мура**. Це пояснюється тим, що він здатний пропускати значні частини тексту, не порівнюючи кожен символ, особливо коли підрядок довгий або не містить частих повторень символів тексту.

Алгоритм **Рабіна-Карпа** зазвичай є найповільнішим у цих тестах через накладні витрати на обчислення хешів.
